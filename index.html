Documentation

Everything you need to get started with SunSet

Getting Started

SunSet is a single-binary media server. The backend (Rust + Axum) serves both the API and the embedded React frontend. Download, run, and you're live.

Installation

Run the install command for your OS:

# Linux / macOS
curl -sL https://sudoloser.github.io/sunset/linux/install.sh | bash
# Windows (PowerShell)
irm https://sudoloser.github.io/sunset/windows/install.ps1 | iex

Manual Setup

Or grab the latest binary from the releases page and run:

./sunset-server

Open it in your browser and follow the setup wizard to create your admin account.

Quick Start

Once the server is running:

  • Create an admin account on first visit
  • Add a library path (e.g. /media/movies)
  • SunSet will scan the directory and fetch metadata automatically
  • Start watching from any device on your network

Library Scanning

SunSet watches your media directories and automatically indexes new files. The scanner runs on startup and can be triggered manually from the admin panel.

How It Works

  • Each library path is scanned recursively for video files
  • Supported formats: mp4mkvavimov
  • Duplicate detection — files already in the DB are skipped
  • Metadata is fetched from TMDB based on the filename
  • Artwork (posters, backdrops, logos) is cached to disk

Adding Libraries

Go to the admin panel and click "Add Library". Enter a name and the absolute path to your media folder. SunSet will start scanning immediately.

# Example library paths
/mnt/storage/movies
/mnt/storage/tv-shows
/media/anime

Movie & Show Parsing

SunSet uses regex-based filename parsing to identify your media and fetch metadata from TMDB. Libraries are tagged as movies or shows when you add them — each type uses a different parser.

Required Folder Structure

Your media must be organized exactly like this:

[movies]
movies
|_'Iron Lung (2026)'
  |_'Iron Lung (2026).mp4'

[shows]
shows
|_'Breaking Bad'
  |_'Season 01'
    |_'Breaking Bad S01E01 Pilot.mp4'

Movie Parsing

Each movie must be in its own subfolder. The filename needs the year in parentheses:

# Regex: ^(.*)\((\d{4})\)$
# Group 1 = title, Group 2 = year

movies
|_'The Matrix (1999)'
| |_'The Matrix (1999).mkv'
|_'Inception (2010)'
| |_'Inception (2010).mp4'
|_'Interstellar (2014)'
  |_'Interstellar (2014).mkv'
  • Only Title (YYYY).ext is parsed — quality tags like 1080p, bluray are not stripped
  • If no year is found, the entire filename stem is used as the search query (worse results)
  • The first TMDB result with a matching year (if found) is selected

TV Show Parsing

Each show in its own folder, episodes inside Season XX subfolders. Filenames must use SxxExx:

# Regex: (?i)^(.*?)\s*S(\d{2})E(\d{2})
# Group 1 = show title, Group 2 = season, Group 3 = episode

shows
|_'Breaking Bad'
| |_'Season 01'
| | |_'Breaking Bad S01E01 Pilot.mkv'
| | |_'Breaking Bad S01E02 Cat's in the Bag.mkv'
| |_'Season 02'
| | |_'Breaking Bad S02E01 Seven Thirty-Seven.mkv'
|_'Mr. Robot'
| |_'Season 01'
|   |_'Mr. Robot S01E01 eps1.0_hellofriend.mkv'
|_'Peaky Blinders'
| |_'Season 01'
|   |_'peaky-blinders-s01e01.mkv'
|_'Some Show'
  |_'Season 00' # specials / extras
    |_'Some Show S00E01 Behind the Scenes.mkv'
  • Only S01E01 / s01e01 is matched — 1x01 and flat folder structures are not supported
  • The show title is everything before SxxExx (trimmed)
  • Episode titles after the episode number are optional (the API fills them in)
  • Metadata (overview, cast, rating, genres, artwork) is fetched per-show from TMDB

Metadata Cache

Fetched metadata and artwork are cached to ~/.sunset/cache/ to avoid repeated API calls. You can trigger a refresh from the media details panel or the admin page.

Database Schema

-- Movies: media_type='movie', year=YYYY
-- Episodes: media_type='episode', show_title + season + episode

media_items
├── id              TEXT   PRIMARY KEY
├── library_id      TEXT   NOT NULL
├── title           TEXT   NOT NULL
├── show_title      TEXT            -- NULL for movies
├── file_path       TEXT   UNIQUE
├── media_type      TEXT   NOT NULL  -- 'movie' | 'episode'
├── year            INTEGER         -- NULL for episodes
├── season          INTEGER         -- NULL for movies
├── episode         INTEGER         -- NULL for movies
├── description     TEXT
├── cast            TEXT
├── genres          TEXT
├── rating          REAL
├── tmdb_id         TEXT
└── added_at        DATETIME

Playback Features

  • Playback speed control (0.5x — 2x)
  • Subtitle track picker with live styling
  • Episode list panel for TV shows
  • Autoplay next episode
  • Cloud-synced progress (resume where you left off)
  • Skip Intro button
  • Touch controls for mobile

Configuration

SunSet is configured through environment variables and the admin panel UI.

Admin Panel

The Admin panel (Settings → Admin) lets you:

  • View storage usage and server stats
  • Generate invite codes for new users
  • Trigger library rescans
  • Remove media items

Subtitle Styling

Configure subtitle appearance under Settings → Subtitles:

  • Text color, size, and font family
  • Background color (CC mode)
  • Bold toggle
  • Live preview while adjusting

Themes

Toggle between dark and light themes in Settings → Appearance.
